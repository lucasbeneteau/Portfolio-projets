# Projet 02 — ITSM & GLPI

## Objectif

Déployer une **solution de gestion de tickets (ITSM)** afin de centraliser les demandes d'assistance informatique et améliorer le suivi des incidents.

Cette solution permet :

- la gestion des **tickets utilisateurs**
- le suivi des **incidents**
- la gestion du **parc informatique**
- l'amélioration du **support IT**

# Architecture du projet

L'infrastructure repose sur une **stack LAMP** :

- **Linux** : système d'exploitation serveur
- **Apache** : serveur web
- **MySQL** : base de données
- **GLPI** : solution ITSM

# Étapes de réalisation

- Installation d’un **serveur Linux** et mise à jour du système
- Installation de la **stack LAMP** (Apache, MySQL, PHP)
- Création de la **base de données GLPI**
- Installation et configuration de **GLPI**
- Création et gestion de **tickets utilisateurs**
- Définition d’un **niveau de priorité des incidents**
- Attribution des tickets à des **techniciens N1 / N2**
- Installation de **GLPI Agent** sur un poste client pour l’inventaire automatique
- Rédaction d'une documentation sur **GLPI Agent**
- Réalisation d’un **logigramme du processus de gestion des tickets**
